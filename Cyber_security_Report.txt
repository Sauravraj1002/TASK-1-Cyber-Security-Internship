Microsoft Remote Procedure Call is used by Windows services for communication and remote management functions. 
Port 139 – NetBIOS 
NetBIOS Session Service enables legacy Windows file and printer sharing functionality. 
Port 445 – SMB 
Server Message Block (SMB) is used for file sharing, printer sharing, and network communication between Windows 
systems. 
Security Risks 
1. Open SMB services may be targeted by ransomware and malware. 
2. NetBIOS can expose network information if improperly configured. 
3. RPC services may become attack vectors if vulnerable software exists. 
4. Unnecessary services increase the attack surface of a device. 
Findings 
1. Two active hosts were identified within the network. 
2. The default gateway was running DNS services on port 53. 
3. The local Windows machine exposed ports 135, 139, and 445. 
4. No unusual or suspicious services were discovered. 
5. Most identified ports are standard Windows networking services. 
3 
Recommendations 
1. Keep Windows Firewall enabled. 
2. Disable file sharing services if not required. 
3. Regularly update Windows and installed software. 
4. Restrict SMB access to trusted networks only. 
5. Monitor open ports periodically using Nmap. 
6. Use strong authentication and network access controls. 
Conclusion 
The network scan successfully identified active hosts and open ports within the local network. The gateway device exposed 
DNS services while the Windows system exposed standard networking and file-sharing ports. Although the discovered 
services are commonly found in Windows environments, unnecessary services should be disabled and firewall protections 
should remain enabled to reduce potential security risks. 
The task provided practical experience in network reconnaissance, port scanning, service identification, and basic security 
assessment using Nmap.
